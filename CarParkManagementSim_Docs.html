Overview v1.0

A JavaFX GUI-based simulation demonstrating automated car park management using the Producer-Consumer threading pattern.

This project models a parking lot as a bounded buffer where Producer threads generate cars, Consumer threads guide them out, and the shared buffer enforces capacity constraints through thread-safe synchronization using Semaphores, Mutexes, Condition Variables, and Peterson's Algorithm.

Producers

Generate cars at configurable rates, simulating arrivals at the parking gate

Buffer (Lot)

Fixed-size bounded buffer protected by Semaphores, Mutex, and Condition Variables

Consumers

Parking guides remove cars, simulate processing work, and release spaces

Built for the Operating System course — Royal University of Phnom Penh, Faculty of Engineering, Department of IT Engineering.

Synchronization Strategy

Producer Flow (parkCar)

SemaphoreemptySlots.acquire() — Block if lot full
↓
Mutexlock.lock() — Enter critical section
↓
Critical Sectionwhile(full) notFull.await()
↓
Shared Statequeue.add(car)
↓
ConditionnotEmpty.signal() — Wake consumer
↓
Mutexlock.unlock() — Exit critical section
↓
SemaphoreoccupiedSlots.release() — Signal occupied

Consumer Flow (removeCar)

Java
// 1. Semaphore: wait for a parked car
occupiedSlots.acquire();

lock.lock();  // 2. Mutex: enter critical section
try {
    while (slots.isEmpty()) {
        notEmpty.await();  // 3. Condition: sleep until car arrives
    }
    car = slots.poll();   // 4. Modify shared state
    notFull.signal();     // 5. Wake a blocked producer
} finally {
    lock.unlock();       // 6. Exit critical section
}

emptySlots.release(); // 7. Signal a slot is free

Peterson's Algorithm

Classical two-process mutual exclusion using only volatile flags and a turn variable. Guarantees mutual exclusion, progress, and bounded waiting without hardware atomics.

Java
public void lock(int threadId) {
    int other = 1 - threadId;

    flag[threadId] = true;   // I want to enter
    turn = other;            // But I'll let you go first

    // Busy-wait: only enters when other doesn't want it
    // OR it's my turn
    while (flag[other] && turn == other) {
        Thread.yield();
    }
    // ═══ CRITICAL SECTION BEGINS ═══
}

public void unlock(int threadId) {
    flag[threadId] = false;  // I'm done
    // ═══ CRITICAL SECTION ENDS ═══
}

Mutual Exclusion

Only one thread in the critical section at a time

Progress

If no thread is inside, one can always enter

Bounded Waiting

A waiting thread will eventually get access

Car Driving Animation

Cars go through a full lifecycle with smooth driving animations:

Drive In

Car appears on the road, drives to the lot entrance (Phase 1 with smoothstep easing), then turns and slides into its assigned slot (Phase 2 with bounce).

Queue When Full

When the lot is full, new cars queue visibly on the road with headlights on and a "X waiting" counter displayed.

Drive Out

When consumed, the car drives from its slot to the lot entrance, then continues down the road and off-screen.

Java — Smoothstep Easing
// Two-phase animation with Hermite interpolation
if (progress < 0.4) {
    // Phase 1: Road → Lot entrance
    double t = progress / 0.4;
    double ease = t * t * (3 - 2 * t);  // smoothstep
    car.wz = lerp(startZ, lotEntrance, ease);
} else {
    // Phase 2: Lot entrance → Parking slot
    double t = (progress - 0.4) / 0.6;
    double ease = t * t * (3 - 2 * t);
    car.wx = lerp(roadX, slotX, ease);
    car.wy = Math.sin(t * Math.PI) * 0.1;  // bounce
}
